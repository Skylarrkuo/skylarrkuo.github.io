从 hexo-admin 的漏洞堆里爬出来，我自己写了一个 Hexo 管理后台

起因

事情是这样的。

我一直在用 hexo-admin 这个插件来管理我的 Hexo 博客。说实话挺好用的，能在浏览器里直接编辑文章，不用每次都开终端敲命令。但问题来了——这东西早就没人维护了。

某天我跑了一下 npm audit，满屏的红色高危漏洞看得我头皮发麻。deep prototype pollution、path traversal、各种 CVE，密密麻麻列了一堆。想着去 GitHub 上看看有没有人修，结果 Issues 区一片荒凉，最近一次提交已经是好几年前的事了。

试着找替代品，要么也是年久失修，要么功能残缺得没法用。Hexo 生态里管理后台这块，基本属于被遗弃的状态。

行吧，自己动手。

要做什么样的东西

既然决定重写，那就好好想想要做成什么样。我的需求其实很明确：

  1. 零外部前端依赖 —— 不搞 npm install 一堆前端框架那一套，直接 Vue 3 + marked.js 走 CDN（后来改成了本地 vendor 目录，后面会说为什么）
  2. 直接调 Hexo 内部 API —— 不绕弯子，直接用 hexo.model() 拿数据，用 hexo.call() 跑命令
  3. 只依赖两个小包 —— hexo-front-matter 解析文章头信息，js-yaml 处理配置文件，完事
  4. 全中文界面 —— 反正主要是自己用，顺便开源给中文社区

功能上，把日常写博客需要的操作都覆盖了：文章增删改查、草稿和发布状态切换、Markdown 实时预览、媒体文件上传、分类标签管理、站点配置编辑、一键 generate/deploy/clean。

技术上的一些考量

为什么不用 Express

Hexo 自己的 server_middleware 钩子就能挂中间件，没必要再引入一个完整的 Express。整个后端就是一个中间件函数，按路由前缀分流——/admin/api 走 API 逻辑，/admin/vendor 提供本地 JS 文件，/admin 返回 SPA 的 HTML。

JWT 认证，但自己实现

没有用 jsonwebtoken 这个包。一个管理后台的认证逻辑用不着那么复杂的库，自己用 Node.js 内置的 crypto 模块写了一个 JWT 签发和验证，总共就几十行代码。HS256 签名，base64url 编码，过期时间检查，够用了。

文件上传：手撕 multipart 解析

这个是整个项目里写起来最麻烦的部分。没有用 multer、busboy 之类的库，直接手动解析 multipart/form-data 的二进制数据。找 boundary、切分 part、解析 header、提取文件内容，全靠 Buffer 操作完成。

说实话写的时候挺痛苦的，但好处是真正的零依赖（除了上面说的两个小包），安装的时候不会拉一堆乱七八糟的东西进来。

前端：一个 HTML 文件搞定

整个前端就一个 Vue 3 的单文件应用，嵌在 html.js 里作为一个模板字符串返回。没有构建步骤，没有 webpack，没有 node_modules 里的前端依赖。

用 hash 路由做页面切换（#/dashboard、#/posts、#/config），每个页面对应一个 Vue 组件区域。编辑器部分用 textarea 写 Markdown，右边实时预览渲染结果。图片支持拖拽上传，分类和标签有自动补全。

用 Claude Code 辅助开发

这个项目是我用 Claude Code 写的，说说感受。

最大的好处是迭代速度快。比如我跟它说”帮我写一个 multipart 解析器，不用任何第三方库”，它直接就把二进制解析逻辑写出来了。虽然第一版有些边界情况没处理好（比如中文文件名、大文件分块），但改几轮就能用。

再比如前端那个 Vue 应用，几千行代码塞在一个模板字符串里，我自己写的话估计得写一个星期。Claude Code 的好处是你描述清楚要什么功能，它就给你生成出来，你再逐个调细节。

但也不是没有坑。有几次它生成的代码逻辑上看着没问题，跑起来才发现有些 Hexo 内部 API 的用法不对（比如 post.categories 的取值方式），得自己对着 Hexo 的源码一个个改。AI 写代码很快，但对特定框架的内部细节理解还是不够深，该查文档查文档，该调试调试，这个省不了。

总结就是：Claude Code 当加速器用很好，但方向盘还是得自己握着。

然后 hexo server，浏览器打开 http://localhost:4000/admin，登录就能用了。

功能一览

  • 仪表盘：文章数、草稿数、分类数、标签数、总字数，一目了然
  • 文章管理：增删改查，支持搜索、按状态筛选、按分类标签过滤，分页浏览
  • 编辑器：Markdown 编辑 + 实时预览，拖拽上传图片，Front Matter 可视化编辑
  • 媒体管理：图片上传、删除、浏览，支持常见图片和视频格式
  • 配置编辑：直接在浏览器里改 _config.yml 和主题配置，支持 YAML 原始编辑和结构化编辑
  • 主题管理：查看已安装主题，切换当前使用的主题
  • 快捷命令：一键生成站点、部署、清除缓存

一些细节

字数统计做了中英文混合处理——中文逐字计数，英文按空格分词。这个看着简单，但对中文博客来说很有必要，纯按空格切的话中文字数永远是 0。

文件名生成支持中文 slug，用了 Unicode 范围匹配 一-鿿，基本上 CJK 统一汉字都能覆盖到。

上传的文件名做了安全过滤，路径穿越（../）之类的攻击在服务端直接拦截了。

写在最后

这个插件说到底就是个工具，解决的是”在浏览器里管理 Hexo 博客”这个具体问题。没有什么宏大的设计理念，代码也不追求什么优雅的架构——能跑、好用、安全，就够了。

如果你也在用 Hexo，也被 hexo-admin 的漏洞问题困扰，可以试试。有问题欢迎提 Issue。
